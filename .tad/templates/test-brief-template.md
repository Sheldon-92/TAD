## 6. 配对测试工作流

标准工作流：
1. 测试 Agent 打开测试环境 URL
2. 按照"测试范围"表格逐页/逐功能测试
3. 每个页面/功能：截图 → 记录问题 → 产品负责人补充反馈
4. 完成所有测试项后，生成汇总报告 (PAIR_TEST_REPORT.md)

### 沟通约定
- 测试 Agent 发现问题时，先描述现象，再等产品负责人判断是否为 Bug
- 产品负责人可以随时补充设计意图，帮助 Agent 理解预期行为
- 不确定的问题标记为"待确认"

## 7. 输出要求

### 截图技术方案

**Web 项目默认**: 使用 gif_creator 方案
- 工具: `window.gif_creator` (浏览器端) 或等效截图工具
- 输出格式: GIF (动态操作录制) 或 PNG (静态截图)

### 截图命名规范
```
e2e-screenshots/{NN}-{page-name}.gif
```
- `NN`: 两位序号 (01, 02, ...)
- `page-name`: 页面/功能的英文短名

### 报告格式要求

输出文件: `PAIR_TEST_REPORT.md`

报告结构:
```markdown
# 配对测试报告

## 测试概览
- 测试日期、时长、覆盖范围

## 测试结果汇总
| 页面/功能 | 状态 | 问题数 | 截图 |
|-----------|------|--------|------|

## 发现的问题
| # | 问题描述 | Priority | 页面 | 截图 | 状态 |
|---|----------|----------|------|------|------|

## 产品反馈
- 设计意图补充
- UX 改进建议
```

## 8. 技术注意事项

### 截图方法对比

| 方法 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| gif_creator | Web 应用 | 动态录制，展示交互流程 | 需要浏览器支持 |
| 原生截图 | 桌面/移动应用 | 通用性强 | 无法录制交互 |
| DevTools | Web 调试 | 可截取完整页面 | 操作复杂 |

### 截图保存示例

```
├── 02-login.gif
├── 03-dashboard.gif
└── 04-settings.png
```

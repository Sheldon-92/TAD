### Pack Collision Detection is an Orthogonal Axis to Per-Pack Eval; Apply the Anti-Theater Spot-Check to the Detector's OWN Bonus Findings - 2026-05-31
- **Context**: Built a cross-pack collision detector (EPIC-pack-collision-detection P1) concurrently with the lean-trustworthy Epic's P5 (per-pack behavioral eval) — two YOLO Conductors in one repo. The two close DIFFERENT gaps: P5 asks "is each pack good ALONE?" (single-pack quality), collision asks "do two CO-LOADED packs contradict?" (pairwise consistency). A pack can pass P5 and still collide with a sibling — independent flags. The detector found 3 real contradictions (Inter font ban vs next/font endorse → cross-category auto-resolve performance>style; APCA vs WCAG 4.5:1 → same-category a11y escalate; testing-pyramid 60% vs 70% → same-category correctness escalate).
- **Discovery**: (1) **Precedence resolves CROSS-category collisions (lower band wins) but MUST escalate SAME-category ones** — a precedence ladder (security>correctness>a11y>performance>style) can only break ties between DIFFERENT bands; two a11y rules (APCA vs WCAG) have no precedence winner, so silent auto-resolve there would fabricate authority. The Inter case (style ban vs performance loading) is the dangerous cross-category one: precedence picks performance, but the resolution is semantically right only because the ban is about PRIMARY typeface while the endorsement is about font-LOADING — precedence gets it right on band-number, not on that distinction, so the VISIBLE LOG carrying the loser's quote is load-bearing for the human spot-check. (2) **A grep collision-scanner is itself validation-theater-prone, so the anti-theater hand-re-derivation must be applied to the detector's OWN "bonus" findings** — the scanner emitted a 4th "bonus genuine" candidate (web-ui-design × video-creation) that the Y6 spot-check proved a FALSE POSITIVE: a CJK `comm -12` keyword-intersection without `LC_ALL=C` produced a phantom shared keyword (设计) between packs that share none. The feature's own thesis (count≠signal, open every file:line) caught a bug in the feature itself. (3) **Two YOLO Conductors can run disjoint-new-file work in one repo safely** with scoped `git add <explicit paths>` (never `-A`) + new-files-only — both Epics committed interleaved to main with zero conflict; worktree merge-back against a concurrently-moving main would have ADDED risk.
- **Action**: For any rule/directive conflict resolver: resolve only across precedence bands, escalate within a band, and make the loser's content visible in the resolution log (precedence decides on band, not semantics). For any detector/scanner feature: run its own anti-theater acceptance (hand-re-derive every flagged item, including bonus/extra findings — those are the most likely false positives). For concurrent multi-agent repo work: scoped-add + disjoint new files beats worktree when files don't overlap.

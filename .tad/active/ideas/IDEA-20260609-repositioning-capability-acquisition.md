# Idea: TAD 重定位 — 能力习得方法论，而非软件开发框架

**ID:** IDEA-20260609-repositioning-capability-acquisition
**Date:** 2026-06-09
**Status:** captured
**Scope:** medium
**Source:** *discuss 对话（唯一真实用户的长期使用体验复盘），2026-06-09

## Summary & Problem

TAD 当前的对外叙事（README / OBJECTIVES）是「软件开发方法论 + Beneficial Friction」，对标对象是 Devin / OpenHands / LangGraph 等编码 agent。但唯一真实用户跨 14 个项目、横跨多领域（播客、声音克隆、硬件、外壳、固件、视频）的长期使用经验表明，**真实价值主张完全不同**：

> **TAD 是一套「习得能力」的方法论：让一个人借助 AI 系统性地进入任何陌生领域，把试错变成阶梯、把探索固化成可复用的能力——而且这些能力最终长在人身上和文档里，不是长在某个工具里。**

定位演进路径（本次对话逐步逼近）：
1. ~~软件开发方法论~~（出生地，不是本质）
2. ~~万能框架~~（表象）
3. 人-AI-文档三角的**复利协议**（机制）
4. **个人能力习得系统**（目的）— 复利的最终标的是人的能力边界

## 核心论点

### 真正的三角不是 Alex–Blake–Human

```
人（意图与判断，但表达有损）
   /                \
AI（能力强大，但无状态 — 每 session 归零）
   \                /
持久化文档层（唯一会复利的东西）
```

裸用 Claude Code / Codex 缺第三个角。用户列举的四个「裸用 AI 做不到」与 TAD 机制一一对应：

| 裸用 AI 工具做不到 | TAD 机制 | 本质 |
|---|---|---|
| 没法一次性把想法说清楚 | 苏格拉底提问（3-5 轮强制） | 意图萃取——承认人的表达有损 |
| 没法制定成功标准 | AC + §9.1 逐行验证 + Gate | 成功的契约化 |
| 失败经验不累积 | Knowledge / principles / AR 注册表 / trace | 失败的结构化沉淀 |
| 没法自然使用自己的方法 | 协议层（角色、handoff、gate） | 方法论的可执行化 |

### 单向棘轮：探索 → 固化 → 调用

**Anchor case（播客，有化石证据）**：声音克隆 + Colab 训练 + TTS 生成 + 改稿 + 配乐——无现成方法论，只能一步步试错。第一次跑通用了 **3 天**（人全程在环，每次失败被记录）；之后同样的事只需 **1 小时**（给稿子 → 全自动 → 验收成品）。那三天的残留物 = `ai-podcast-production` pack（large-chunk TTS、dual-BGM envelope ducking、Colab deployment 判断规则）。

- 压缩比 3天→1小时 就是「把探索编译成能力」的编译收益
- 收敛后人只留在两端（给意图、验结果）= Beneficial Friction 理论的实践自洽
- 棘轮只进不退：普通 AI 用法每个 session 重新踩坑；TAD 把踩坑变成永久资产

### 为什么必须是 CLI 而非 App（结构性原因）

- App 形态的 AI 没有「家」放方法论——每次对话是租来的房间，经验无处常驻
- CLI agent 是 booted into 用户的文件系统：方法论是它的操作系统，不是每次口头重复的提示词
- 探索-固化-调用循环只在「AI 能读写持久层 + 能操作真实环境」同时成立处转得起来

公式：**CLI agent（能执行、能读积累） + TAD（强制每次交互留下结构化残留） = 把一次性探索变成永久能力**

### 确定性的精确含义

TAD 不让 AI 输出确定，让**过程收敛**确定：失败被 gate 抓住、被记录、回收为下次先验 → 到达可接受结果的迭代次数单调下降。是收敛的确定性，不是输出的确定性。

### 三方共同进化，缺一不可

- **AI** 获得可复用执行路径（pack / workflow / skill）
- **文档** 获得结构化经验记忆（knowledge / principles / handoff 史）
- **人** 获得真实领域理解和判断力（与「AI 替你做事、你越来越不需要懂」的行业叙事相反）

护城河不在框架代码（可复制），在积累的语料层（化石层，复制不走）。

## Implications（定位变更的下游影响）

1. **README 叙事重写候选**：从「软件开发方法论」改为「能力习得方法论」；播客 3天→1小时 案例做主叙事
2. **OBJECTIVES O1 对标对象调整**：竞品不（只）是 Devin/OpenHands，而是「外行如何安全指挥 AI 做内行事」赛道（目前近乎空白）
3. **优先级重排信号**：
   - ↑ Pack 行为评估补完（13 个 pending）——陌生领域里 pack 的错误会被用户直接继承
   - ↑ 自动沉淀回路修复（self-evolution loop 0% close-rate）——共同进化是核心主张，这条回路必须闭合
   - ↑ 陌生领域的需求引导（反向苏格拉底：用户不懂时，提问应教用户该问什么）

## 收敛版 — 2026-06-09 讨论 + 研究后（以上初稿多处被推翻，以本节为准）

> 上面是初稿。当天做了一轮深度研究（`.tad/evidence/research/repositioning-3-walls/`，8 个 NotebookLM ask + Codex/Gemini 两轮对抗）+ 一轮 *discuss，结论大幅收敛。**初稿的「赛道空白 / 第三个角稀缺 / 人习得能力」基本被证伪或证据不足**，但塌出来一个更真、更具体的定位。

### 初稿被推翻/收敛的部分
1. **「赛道空白」**——错。但错在测量轴：第一轮测「有没有人定位成 X」（存在性），结论是赛道挤满了（Lovable/Replit/Dify/GitAgent…）。换成深度轴重测才对：所有竞品都钉死在**浅水区**（任务型、well-defined）；复杂/模糊/长程问题上**自主 AI 集体崩溃**（SWE-bench 72.8% → SWE-EVO 25%；嵌套工具调用 28%；10 跳复合→81%）。**深水区实证为空**。
2. **「第三个角（持久文档层）稀缺」**——错。2026 有 9 种持久层机制（CLAUDE.md/Cursor/mem0/Letta/DiffMem/GitAgent…），是标配不是稀缺。「强制留残留」别人也做了。
3. **「人习得能力 / 外行变内行」**——证据不足。没有证据表明 AI 工作流产生「永久、独立的人类技能」；只有实时 augmentation + 跨域**操作**迁移（"everyone becomes more full-stack"）。"看别人举铁不长肌肉。"

### 收敛后真正站得住的定位（有证据 + 真做过）
- **行业正收敛到 TAD 的架构**：深水区 SOTA 答案 = 人机协同编排（LangGraph durable execution、HULA、agent 主动求助）。TAD 的人类编排者 + 强制暂停 + 苏格拉底 + handoff/Gate **正是这个方向 → 验证而非证伪**。Collaboration Paradox（人解模糊那部分，只能完全委派 0-20%）就是 TAD 的论点本身。
- **唯一杠杆是编程，不是"领域通用魔法"**（避免大词）：能用代码够到的领域就能进。播客靠写 VAD/训练/TTS 代码进去，不是"懂声音"。
- **最值钱的具体机制 = 判断界面（judgment surface）**：在**没有自动评判器**的领域（声音/配乐/视频/硬件，"好不好"靠人品味），用编程杠杆**现造一个一次性 HTML 判断界面**（A/B 播放器、打分网格），把人的品味**编译成结构化数据 + 沉淀**（化石证据：Colin 项目 `ab_votes.json`）。这是聊天式 AI 做不到、CLI agent 才能做的——把 idea 文档里「为什么必须 CLI」从口号变成真发生过的能力。
- **价值与领域「可自动评判程度」成反比**：编程最可验证→TAD 边际价值最小；声音/视频/硬件最不可验证→TAD 价值最大。TAD 最该去的恰是 AI 最弱的领域。

### 一句话定位（替代初稿大词）
> TAD = **面向「无自动评判器」领域的人机协同操作系统**。唯一杠杆是编程；最值钱的用法是用代码现造「判断界面」，把人的判断变成高带宽、结构化、可沉淀的信号。不是"外行变内行"，是"有判断力的人能跨陌生领域 ship，且每遍更省"。

### 收敛后的下一步（已与人类确认）
- **现在做（生成性）**：把「**无自动评判器 + 正要张嘴问'你觉得怎么样' → 改成现造判断界面**」这个**反射**固化进核心方法论层；从 playground + 声音 HTML 两个化石把"好的判断界面长什么样"的手艺记一次。
- **等第三个领域再做（避免过早抽象，rule-of-three）**：把 `/playground` 从"视觉专用"降级为该反射的视觉实例，与声音实例共用薄机制。
- **README 重写**：**暂缓**——在「TAD 在困难问题上真的更强」拿到证据前，不把未验证主张刻进对外叙事。

### 仍未证明（诚实保留）
- TAD 方法论在困难问题上是否**真的比随手 human+AI 走得更远**（无 head-to-head 证据 — TAD 至今缺的 eval）。
- 机械门 ROI（代码强制 vs 软提醒，单人工作流值不值）；残留 Staleness Trap（半衰期→幻觉锚，需自主废弃机制，接 Tool Freshness Problem）。

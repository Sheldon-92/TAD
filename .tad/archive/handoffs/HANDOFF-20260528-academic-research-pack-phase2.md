## 1. Task Overview

### 1.1 What We're Building
The core `academic-research` capability pack: a main SKILL.md router + 5 protocol reference files + install.sh. This is the methodology layer — it teaches TAD agents HOW to do research (depth enforcement, citation integrity, quality scoring), NOT what databases to query (Phase 3-4).

### 1.2 Why We're Building It
After this Phase, any TAD project that installs the pack gets a research-quality protocol built from ScienceClaw's 629-line SCIENCE.md + ScholarEval rubric + Reflexion Cycle. The agent will know research phases, enforce citation integrity, and self-evaluate quality — even before Phase 3 adds domain-specific skills.

### 1.3 Intent Statement
**真正要解决的问题**: TAD agents currently have zero guidance for academic research methodology. This pack gives them the judgment rules a trained postdoc applies automatically.

**不是要做的**:
- ❌ NOT database API templates (Phase 3-4)
- ❌ NOT individual skill content (literature-search, meta-analysis, etc. — Phase 3)
- ❌ NOT multimodal image analysis (Phase 5)
- ❌ NOT memory system integration (Phase 5)

### 4.2 SKILL.md Router Design

The SKILL.md should:
1. Have frontmatter: `name: academic-research`, `description:` with UNIQUE keywords that do NOT collide with existing `research-methodology` pack.
   - ⚠️ COLLISION AVOIDANCE: `research-methodology` pack already claims: `研究`, `research`, `调研`, `landscape`, `对比研究`, `deep research`, `分析`, `综述`, `竞品`, `市场调研`, `技术调研`
   - academic-research MUST use domain-specific keywords ONLY: `学术`, `academic`, `论文`, `paper`, `文献`, `literature`, `meta-analysis`, `元分析`, `PRISMA`, `systematic review`, `系统性综述`, `PubMed`, `文献综述`, `学术研究`, `科研`
   - Include disambiguation block in SKILL.md: "If this looks like market research, technology landscape, or competitive analysis → defer to research-methodology pack"
2. Detect research task type from user input:
   - Quick factual → light protocol (single session, ≤5 tool calls)
   - Literature survey → standard protocol (1-2 sessions, 20-40 tool calls)
   - Comprehensive/systematic review → full protocol (Epic phases, 40-100+ tool calls)
   Detection signals: keyword matching + task description analysis (Blake designs the signal-to-tier mapping table, with at least 3 example user inputs per tier)
3. Load appropriate references based on task type
4. Define Alex/Blake role mapping for research tasks
5. Include anti-premature-conclusion checklist (extract all rules from SCIENCE.md, number them)
6. Include CONSUMES/PRODUCES declaration: CONSUMES: research question + optional domain constraints; PRODUCES: evidence-grounded research report + methodology section + citation audit trail

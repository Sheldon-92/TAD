## 1. Task Overview

### 1.1 Executive Summary
为 TAD 能力包安装引入哈希清单基础设施（`.tad-pack-meta.yaml`），记录每个 pack 文件在安装时的 SHA-256 哈希。同时修复 sync-protocol.md 中残留的 `rm -rf` 和 `install.sh` 引用。这是 Pack Content Protection 四阶段 Epic 的基础——后续 Phase 依赖此哈希清单实现修改检测、冲突解决和 fork 支持。

### 1.2 Background
voice-studio 项目在 v2.30.0 sync 后，ai-podcast-production pack 的 3 个项目特有引用文件被删除、tts-production.md 被降级（chunk 100→200-350）。根因是 sync 脚本做 `rm -rf` + `cp -R`（先删后拷），以及 install.sh 从 stale `.tad/capability-packs/` 再生内容。

tad.sh 的 `cp -r`（merge）行为已经安全（不删额外文件），但无法区分"用户没改过"和"用户改过"的文件——后续 Phase 2 需要哈希基础才能做这个判断。

### 1.3 Intent Statement
本 Phase 只做基础设施：安装时记录哈希 + 修复协议文档。不做修改检测逻辑（Phase 2）。

## 3. Technical Design

### 3.1 `.tad-pack-meta.yaml` 格式

```yaml
# Auto-generated by tad.sh / install.sh — do not edit manually
installed_version: "2.31.1"
installed_date: "2026-06-17"
sync_policy: upstream  # upstream | forked (Phase 4)
baseline_source: fresh_install  # fresh_install | migrated (Phase 2 treats migrated as "unknown, preserve")
files:
  - path: SKILL.md
    sha256: "abc123def456..."
  - path: references/tts-production.md
    sha256: "789ghi012jkl..."
  - path: references/music-arrangement.md
    sha256: "345mno678pqr..."
```

### 3.2 tad.sh 修改点

在 `copy_framework_files` 函数中，skill 复制循环（现 line 444-461）之后，添加 meta 生成逻辑：

```bash
# --- Pack meta generation (Phase 1: hash manifest) ---
# Generate .tad-pack-meta.yaml for each installed pack skill.
# Runs for primary platform AND secondary ("both" platform) to keep parity.
if [ -f "$src/.tad/capability-packs/pack-registry.yaml" ]; then
    local meta_targets="$TARGET_SKILL_DIR"
    # "both" platform: also generate meta for secondary skill dir
    [ "$PLATFORM" = "both" ] && meta_targets="$TARGET_SKILL_DIR .agents/skills"
    local mt
    for mt in $meta_targets; do
        [ -d "$mt" ] || continue
        for skill_dir in "$mt"/*/; do
            [ -d "$skill_dir" ] || continue
            local sn
            sn="$(basename "$skill_dir")"
            is_pack_skill "$sn" "$src" || continue
            generate_pack_meta "$skill_dir" || log_warn "Meta generation failed for $sn, skipping"
        done
    done
fi
```

新增 `generate_pack_meta` 函数：

```bash
generate_pack_meta() {
    local skill_dir="$1"
    # ⚠️ P0 fix: normalize trailing slash (glob produces "dir/", stripping needs "dir")
    skill_dir="${skill_dir%/}"
    local meta_file="$skill_dir/.tad-pack-meta.yaml"
    # ⚠️ P0 fix: use $TARGET_VERSION (global, authoritative) — NOT version.txt
    # (version.txt may not exist on fresh install, or hold OLD version on upgrade)
    local version="$TARGET_VERSION"
    local today
    today="$(date +%Y-%m-%d)"

    # Portable SHA-256 command (macOS=shasum, Linux=sha256sum)
    local sha_cmd
    if command -v shasum >/dev/null 2>&1; then
        sha_cmd="shasum -a 256"
    else
        sha_cmd="sha256sum"
    fi

    # Preserve existing sync_policy if meta already exists (Phase 4 forward-compat)
    local existing_policy="upstream"
    local existing_baseline="fresh_install"
    if [ -f "$meta_file" ]; then
        local ep
        ep="$(grep '^sync_policy:' "$meta_file" 2>/dev/null | sed 's/sync_policy:[[:space:]]*//' | tr -d '[:space:]"')"
        [ -n "$ep" ] && existing_policy="$ep"
    else
        # No prior meta = migration from pre-Phase-1 install
        existing_baseline="migrated"
    fi

    {
        printf '# Auto-generated by tad.sh — do not edit manually\n'
        printf 'installed_version: "%s"\n' "$version"
        printf 'installed_date: "%s"\n' "$today"
        printf 'sync_policy: %s\n' "$existing_policy"
        printf 'baseline_source: %s\n' "$existing_baseline"
        printf 'files:\n'
        find "$skill_dir" -type f -not -name '.tad-pack-meta.yaml' -not -path '*/local/*' | sort | while read -r f; do
            local rel
            rel="${f#"$skill_dir"/}"
            local hash
            hash="$($sha_cmd "$f" 2>/dev/null | cut -d' ' -f1)" || continue
            printf '  - path: "%s"\n' "$rel"
            printf '    sha256: "%s"\n' "$hash"
        done
    } > "$meta_file"
}
```

**⚠️ 错误处理**：整个 `generate_pack_meta` 调用应包裹在 `|| log_warn` 中，避免 meta 生成失败触发 `set -euo pipefail` 的 ERR trap 导致整个安装回滚：
```bash
generate_pack_meta "$skill_dir" || log_warn "Meta generation failed for $sn, skipping"
```

### 3.3 install.sh 模板修改

在文件末尾（`=== Installation complete ===` 之前），添加 meta 生成逻辑。

**版本来源**：install.sh 没有 `$TARGET_VERSION`。方案：读 `../../.tad/version.txt`（install.sh 的 CWD 是项目根），fallback 到 `"standalone"`。

```bash
# --- Pack meta generation ---
local_version="$(cat .tad/version.txt 2>/dev/null | tr -d '[:space:]')"
[ -z "$local_version" ] && local_version="standalone"
local_sha_cmd
if command -v shasum >/dev/null 2>&1; then local_sha_cmd="shasum -a 256"; else local_sha_cmd="sha256sum"; fi

local_baseline="fresh_install"
[ -f "${TARGET_DIR}/.tad-pack-meta.yaml" ] && local_baseline="fresh_install" || local_baseline="fresh_install"
# install.sh always creates fresh — no migrated state

{
  printf '# Auto-generated by install.sh — do not edit manually\n'
  printf 'installed_version: "%s"\n' "$local_version"
  printf 'installed_date: "%s"\n' "$(date +%Y-%m-%d)"
  printf 'sync_policy: upstream\n'
  printf 'baseline_source: fresh_install\n'
  printf 'files:\n'
  find "${TARGET_DIR}" -type f -not -name '.tad-pack-meta.yaml' -not -path '*/local/*' | sort | while read -r f; do
    rel="${f#"${TARGET_DIR}"/}"
    hash="$($local_sha_cmd "$f" 2>/dev/null | cut -d' ' -f1)" || continue
    printf '  - path: "%s"\n' "$rel"
    printf '    sha256: "%s"\n' "$hash"
  done
} > "${TARGET_DIR}/.tad-pack-meta.yaml"
```

**注意**：install.sh 中不用 `local` 关键字（不在函数内），改用 `local_` 前缀变量名避免全局污染。

## 4. Implementation Guide

### 4.1 Step-by-step

1. **tad.sh**: 在 `copy_framework_files` 末尾（skill 复制循环之后），添加 `generate_pack_meta` 函数和调用逻辑
2. **tad.sh**: `generate_pack_meta` 函数定义放在 `is_pack_skill` 函数附近（约 line 314-324）
3. **install.sh template**: 在安装完成后生成 meta（类似逻辑，但用 `$TARGET_DIR` 和 `$PACK_NAME`）
4. **sync-protocol.md**: 删除 step b2 整个段落，修改 skill copy 描述

### 4.2 Edge Cases

- **空 pack**（无 reference 文件）：meta 的 files 列表只有 SKILL.md
- **非 pack skill**（alex, blake, gate 等）：`is_pack_skill` 返回 false → 不生成 meta
- **`--packs` 选择安装**：只为选中的 pack 生成 meta；已安装但未选中的 pack 保留原 meta 不变
- **已有 meta 的 pack**：覆盖 meta 但保留 sync_policy（Phase 4 forward-compat），baseline_source 重置为 fresh_install
- **无 pack-registry.yaml**（非 TAD 源安装）：跳过 meta 生成
- **`both` 平台**：为 `.claude/skills/` 和 `.agents/skills/` 两侧都生成 meta
- **首次迁移（无 meta）**：baseline_source 设为 `migrated`，Phase 2 将视为"unknown, preserve"而非 pristine
- **shasum 失败**（权限问题/损坏符号链接）：单文件 `|| continue` 跳过，整函数 `|| log_warn` 防止回滚

## 10. Decision Summary

| Decision | Choice | Rationale |
|----------|--------|-----------|
| 哈希算法 | SHA-256 (shasum/sha256sum with portable detection) | macOS/Linux 自带，无需额外依赖 |
| Meta 文件位置 | pack skill 目录内（.tad-pack-meta.yaml） | 跟随 pack 移动，无需外部注册表 |
| 首次迁移策略 | 当前文件作为 baseline + baseline_source: migrated 标记 | Phase 2 区分 migrated（preserve）vs fresh_install（safe to overwrite） |
| local/ 排除 | find -not -path '*/local/*' | 为 Phase 2+ 的项目保护空间预留 |
| sync-protocol 修复 | 删除 install.sh 步骤 | v2.30.0 已验证 cp -R mirror 是正确方式 |
| 版本来源 | tad.sh 用 $TARGET_VERSION；install.sh 读 version.txt fallback "standalone" | 避免时序错误（version.txt 在 copy 后才写入） |
| 错误处理 | 单文件 continue + 整函数 log_warn | meta 是基础设施，失败不应回滚整个安装 |

✅ Epic COMPLETE (2026-07-03, 2/2 phases in single session):
- Phase 1: gbrain POC → 1/5 FAIL (NEGATIVE-RESULT). BM25 without embeddings ≈ grep.
  gbrain's value is in LLM features (embedding + think synthesis), not mechanical (BM25 + entity graph).
  Entity graph = 0 links (TAD doesn't use wikilinks). Import was flawless (2282 files, 50s, 0 errors).
- Phase 2: TAD-native → 5/5 PASS. Claude IS the semantic search engine. 250-line index (`brain-index.md`)
  generated by bash script (`brain-index-gen.sh`) helps Claude locate files, then general-purpose agent
  reads + synthesizes. Zero external dependencies.

**Why:** Knowledge search reimagined after Phase 1 showed external tools add value only through LLM features.
**How to apply:** When evaluating external tools, ask "does this tool's value come from something Claude already does?"
  If yes, a thin wrapper around Claude's native capabilities beats importing the tool's full stack.

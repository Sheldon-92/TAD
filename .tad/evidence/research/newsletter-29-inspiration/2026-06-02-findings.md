## Key Finding 1: 73% Token Waste Is Real and Solvable

ArgosBrain measured that **73% of AI coding agent tokens are redundant re-reads**. The arxiv paper (2603.27277) independently verified **99.2% token reduction** (3,400 vs 412,000 tokens) for structural queries via graph vs file-by-file grep.

**Architecture**: Two-pass indexing (tree-sitter syntactic → hybrid LSP semantic), persisted to SQLite, queried via MCP. 14 tools including `detect_changes` (git diff → blast radius) and `trace_call_path` (BFS caller traversal).

**TAD impact**: TAD's `step1c_lsp` runs LSP per-handoff as a one-shot analysis. codebase-memory-mcp replaces this with persistent graph queries that survive across handoffs. The `detect_changes` tool directly replaces TAD's documentSymbol + incomingCalls chain.

---

## Key Finding 2: In-Session Learning Is the Gap TAD Doesn't Fill

TAD's learning loop is cross-session: trace events → `*optimize` batch analysis → proposals → human approval → next session. This misses **in-session real-time learning** — when Blake hits a failing gate due to a project-knowledge rule, it can't query *why* that rule exists.

**DiffMem's approach**: Expose `git log`, `git diff`, `git blame` as sandboxed tools. The agent queries its own knowledge history on-demand. Current state files stay lean; history is pulled only when needed.

**Concrete TAD integration**: Give Blake a `git-blame-knowledge` MCP tool scoped to `.tad/project-knowledge/`. When Blake encounters a rule that seems wrong, it can `git blame` the specific line to find when/why it was added (commit message = the handoff that discovered it), then make an informed decision vs blindly following or ignoring.

## Key Finding 3: consulting-os Is TAD's Shadow Competitor

Sean Cofoid's consulting-os (git-versioned agent memory + role-separated skills + session-inline meta-heuristic learning) maps nearly 1:1 to TAD:

| Dimension | TAD | consulting-os |
|-----------|-----|---------------|
| Memory | .tad/project-knowledge/*.md | git-versioned plain files |
| Roles | Alex/Blake (terminal isolation) | role-separated skills |
| Learning | *optimize/*evolve (cross-session) | plan→debug→regression→meta-heuristic (in-session) |
| Verification | Gate 3/4 + Knowledge Assessment | "falsifiable self-improvement" |
| Models | Claude family | claude-sonnet-4-6 + opus-4-7 |

**Key difference**: consulting-os meta-heuristic runs inline (same session), TAD's runs offline. consulting-os is tighter but narrower; TAD's cross-project *evolve is broader.

## Key Finding 4: GitAgent Protocol (GAP) as an Export Standard

GitAgent defines a portable agent format: `agent.yaml` + `SOUL.md` + `RULES.md` + `memory/` + `skills/` + `hooks/`. It exports to Claude Code, Codex, Gemini CLI, and 8+ frameworks.

**TAD relevance**: TAD's `.tad/` directory structure is already close to GAP. The main gap is TAD lacks a formal `agent.yaml` manifest. Exporting TAD as a GAP-compatible repo would enable portability to non-Claude runtimes.

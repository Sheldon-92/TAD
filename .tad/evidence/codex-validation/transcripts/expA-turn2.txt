## 1. Executive Summary

Add a Bash function named `truncate_slug` to:

`.tad/evidence/codex-validation/sandbox/truncate_slug.sh`

The function accepts two arguments:

1. `slug`
2. `max_length`

It prints a slug truncated to at most `max_length`, preferring the last hyphen before the limit so words are not split. If no such hyphen exists, it hard-truncates at `max_length`.

## 2. Requirements

- Function name: `truncate_slug`
- File path: `.tad/evidence/codex-validation/sandbox/truncate_slug.sh`
- Input: `truncate_slug "<slug>" "<max_length>"`
- Output: result printed to stdout
- Invalid max length:
  - Print an error to stderr
  - Return non-zero
- Do not execute behavior on source unless intentionally invoked by caller.

## 3. Behavior Rules

- If `slug` length is `<= max_length`, return unchanged.
- If `max_length` is `0`, return empty.
- If `slug` is empty, return empty.
- If `max_length` is non-numeric, print error to stderr and return non-zero.
- If truncation is needed:
  - Inspect the prefix of `slug` up to `max_length`.
  - If that prefix contains a hyphen, truncate at the last hyphen before or at the limit.
  - Exclude the trailing hyphen from the returned value.
  - If no hyphen exists before the limit, hard-truncate to exactly `max_length`.

## 4. Files To Modify

# TAD Codex Adapter — Migration Notice

> As of TAD v2.26.0, Codex uses the **same unified SKILL.md files** as Claude Code.

## What Changed

The separate "compressed" Codex SKILL editions (`codex-alex-skill.md`, `codex-blake-skill.md`)
and all supporting scripts have been removed. Codex now receives the full SKILL.md files
installed to `.agents/skills/` (the official Codex skill path).

## How It Works Now

- `bash tad.sh --platform codex --yes` installs skills to `.agents/skills/`
- AGENTS.md triggers Alex/Blake via `.agents/skills/{alex,blake}/SKILL.md`
- Hooks are configured in `.codex/hooks.json` (auto-generated by tad.sh)
- All 24+ capability packs install to `.agents/skills/` on Codex

## Files Removed in v2.26.0

- `codex-alex-skill.md` / `codex-blake-skill.md` (compressed SKILL editions)
- `codex-tad-alex.sh` / `codex-tad-blake.sh` (launcher scripts)
- `regen-codex-editions.sh` (regeneration script)
- `tournament-codex.sh` / `schemas/` (tournament adapter)
- `expert-review-sequential.md` / `sequential-review.md` / `manual-gates.md` / `socratic-fallback.md` (adapter guides)
- `.tad/hooks/lib/codex-parity-check.sh` (parity enforcement)

This README is the only file retained in `.tad/codex/` for migration reference.

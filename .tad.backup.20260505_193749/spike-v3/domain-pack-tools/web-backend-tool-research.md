## 1. Prisma CLI (ORM / Database)

**Status: PASS** — Excellent fit for domain pack

### Install
```bash
npm install -D prisma@6.5.0
```

### Tests

| Command | Result | Notes |
|---------|--------|-------|
| `npx prisma init --datasource-provider sqlite` | PASS | Creates `prisma/schema.prisma` + `.env` |
| `npx prisma validate` | PASS | Validates schema syntax and relations |
| `npx prisma format` | PASS | Auto-formats schema file |
| `npx prisma generate` | PASS | Generates client WITHOUT a running DB |
| `npx prisma db push` | PASS | Creates SQLite DB from schema (no server needed) |

### Key Findings
- **No Docker required** for SQLite provider
- `prisma validate` + `prisma format` are pure file operations — ideal for Claude Code
- `prisma db push` with SQLite creates a local file DB — zero infrastructure
- `prisma generate` produces typed client code without any DB connection
- For Postgres/MySQL: needs a running DB for `db push` / `migrate`, but `validate` / `format` / `generate` still work offline

### Generators (Bonus)

**prisma-erd-generator** — Generates Mermaid ER diagrams from Prisma schema:
```bash
npm install -D prisma-erd-generator@1.11.2 --legacy-peer-deps
```
Add to schema.prisma:
```prisma
generator erd {
  provider = "prisma-erd-generator"
  output   = "../erd.md"
}
```
Result: produces valid Mermaid `erDiagram` with entities, fields, keys, and relationships. **PASS**.

**zod-prisma-types** — Generates Zod schemas from Prisma schema (see Zod section below).

### Domain Pack Usage
- `prisma init --datasource-provider sqlite` — scaffold DB schema
- `prisma validate` — validate schema correctness
- `prisma format` — format schema file
- `prisma generate` — generate client + zod types + ER diagram
- `prisma db push` — apply schema to SQLite (for prototyping)

## 2. Supabase CLI

**Status: PARTIAL PASS** — `init` works, `start` requires Docker

### Install
```bash
npx supabase@2.20.0 --version
# Output: 2.20.0
```

### Tests

| Command | Result | Notes |
|---------|--------|-------|
| `npx supabase --version` | PASS | Installs and runs via npx |
| `npx supabase init` | PASS | Creates `supabase/` directory with config |
| `npx supabase start` | FAIL | **Requires Docker daemon** |
| `npx supabase gen types` | FAIL | Needs `--local` (Docker) or `--linked` (remote project) |

### Key Findings
- **Docker is REQUIRED for local development** (`supabase start`, migrations, local DB)
- `supabase init` is the only useful offline command — creates config scaffold
- `supabase gen types --linked` works with a remote Supabase project (needs auth token)
- For domain pack: **limited utility** without Docker. Can scaffold project structure but cannot run locally

### Domain Pack Recommendation
- Include only for **project scaffolding** (`init`) and **remote type generation** (if project ID available)
- Mark as "requires Docker for full functionality" in tool metadata
- Lower priority than Prisma for offline/local development

## 6. Zod (Runtime Validation)

**Status: PASS** — Standalone + Prisma integration both work

### Install
```bash
npm install -D zod@3.24.0
```

### 6a. Standalone Validation

```javascript
const { z } = require('zod');
const UserInput = z.object({
  name: z.string().min(1),
  email: z.string().email(),
  age: z.number().int().positive().optional(),
});

// Valid: { success: true, data: { name: 'Alice', email: 'alice@example.com', age: 30 } }
// Invalid: { success: false, errors: ["String must contain at least 1 character(s)", "Invalid email"] }
```

### 6b. Prisma → Zod Generation

```bash
npm install -D zod-prisma-types@3.1.8
```

Add to `schema.prisma`:
```prisma
generator zod {
  provider = "zod-prisma-types"
  output   = "../src/zod"
}
```

`npx prisma generate` produces:
```typescript
export const UserSchema = z.object({
  id: z.number().int(),
  email: z.string(),
  name: z.string().nullable(),
  createdAt: z.coerce.date(),
});
export type User = z.infer<typeof UserSchema>;
```

### Key Findings
- `zod-prisma-types` auto-generates Zod schemas from Prisma models — single source of truth
- Handles nullable fields, relations, enums correctly
- Generated schemas include `z.infer<>` type exports
- **Prisma → Zod pipeline works with zero manual effort**: edit schema.prisma → `prisma generate` → get both client AND validation schemas

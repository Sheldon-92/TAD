# Meeting Notes & Minutes Skill

> 综合自会议管理最佳实践，已适配 TAD 框架

## 触发条件

当用户需要记录会议、整理会议纪要、提取行动项或生成会议摘要时，自动应用此 Skill。

---

## 核心能力

```
会议记录工具箱
├── 会议准备
│   ├── 议程制定
│   ├── 参会邀请
│   └── 材料准备
├── 实时记录
│   ├── 要点捕捉
│   ├── 决策记录
│   └── 行动项识别
├── 会后整理
│   ├── 纪要撰写
│   ├── 摘要生成
│   └── 任务分配
└── 跟进追踪
    ├── 行动项追踪
    ├── 决议执行
    └── 会议回顾
```

## 会议记录技巧

### 记录要点

```markdown
## 高效记录技巧

### 记什么
✅ 决策和结论
✅ 行动项（谁、做什么、何时完成）
✅ 关键论点和理由
✅ 不同意见和争议点
✅ 需要后续跟进的问题

### 不记什么
❌ 逐字记录每句话
❌ 无关的闲聊
❌ 重复的讨论内容
❌ 已在文档中记录的背景信息

### 记录符号

| 符号 | 含义 |
|------|------|
| ✓ | 已达成共识 |
| ? | 需要确认 |
| ! | 重要/紧急 |
| → | 行动项 |
| ☆ | 好主意/重点 |
| ⚠ | 风险/警告 |
```

```markdown
- [ ] **[任务标题]**

- [ ] @张三: [任务描述] (截止: MM-DD)

**带状态的格式**:
| 任务 | 负责人 | 截止日期 | 状态 |
|------|--------|----------|------|
| [任务] | @张三 | MM-DD | 🔴 未开始 |

| [任务] | @王五 | MM-DD | 🟢 已完成 |
```

```markdown
## 行动项状态
```

## 会议摘要生成

### 摘要提取规则

```markdown
## 从会议记录生成摘要

### 输入
[粘贴会议记录或转录文本]

### 提取要素

1. **会议基本信息**
   - 识别日期、参与者、主题

2. **关键决策**
   - 提取明确做出的决定
   - 标注决策的背景和理由

3. **行动项**
   - 识别"谁要做什么"的承诺
   - 提取截止日期

4. **重要讨论**
   - 总结主要观点
   - 记录争议和不同意见

5. **后续安排**
   - 下次会议时间
   - 待解决问题

### 输出格式
[使用上述会议摘要模板]
```

## 最佳实践

```
✅ 推荐
□ 会前发送议程，会后24小时内发送纪要
□ 使用标准模板保持一致性
□ 明确记录决策和行动项
□ 指定具体负责人和截止日期
□ 定期回顾和更新行动项状态

❌ 避免
□ 会议没有明确目标和议程
□ 纪要过于冗长或过于简略
□ 行动项没有负责人或截止日期
□ 会后不跟进行动项执行
□ 重要决策没有记录理由
```

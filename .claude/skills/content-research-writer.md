## TL;DR Quick Checklist

```
1. [ ] 研究：主题/受众/关键词/竞品
2. [ ] 规划：大纲/结构/语气/示例
3. [ ] 写作：AIDA/PAS/FAB 结构与可读性
4. [ ] 审校：事实核验/引用/语法/风格
5. [ ] 产出：来源清单/大纲/草稿版本
```

**Red Flags:** 无受众定位、无大纲、抄袭、堆关键词、无引用

当用户需要进行内容研究、撰写文章、创作博客、制作营销文案或其他内容创作任务时，自动应用此 Skill。

## 标题写作技巧

### 标题公式

```
1. 数字型
   "7 个方法让你的 [X] 提升 200%"
   "2024 年必须掌握的 10 个 [X] 技巧"

2. 问题型
   "为什么你的 [X] 总是失败？"
   "你知道 [X] 的真正含义吗？"

3. 指南型
   "[X] 完全指南：从入门到精通"
   "新手必看：[X] 入门实战手册"

4. 对比型
   "[A] vs [B]：哪个更适合你？"
   "正确 vs 错误：[X] 的 5 个常见误区"

5. 时效型
   "2024 年 [X] 最新趋势解读"
   "[X] 有什么新变化？更新版指南"

6. 结果型
   "我如何在 30 天内掌握 [X]"
   "这个方法让我的 [X] 提升了 3 倍"
```

### 标题优化清单

```
□ 包含主关键词
□ 字数控制在 15-25 字
□ 包含数字或具体承诺
□ 能激发好奇心或利益点
□ 避免标题党，内容要兑现
□ 适配平台特点（微信/知乎/头条）
```

## 写作技巧

### 开头吸引力法则

```markdown
好的开头方式:

1. 痛点共鸣
   "你是否也遇到过这样的情况：[描述痛点场景]"

2. 数据冲击
   "数据显示，80% 的人都犯了这个错误..."

3. 故事引入
   "去年，我遇到了一个难题..."

4. 反常识观点
   "你以为的 [X] 其实是错的"

5. 直接利益
   "读完这篇文章，你将学会..."
```

### 段落结构

```
PREP 法则:
- Point (观点): 开门见山说出核心观点
- Reason (理由): 解释为什么这样说
- Example (例子): 用案例或数据支撑
- Point (重申): 总结强化观点
```

### 可读性优化

```markdown
✅ 推荐做法:
□ 段落不超过 5 行
□ 使用子标题分割内容
□ 重点内容加粗或高亮
□ 使用列表呈现并列信息
□ 适当使用图片/图表
□ 添加引用框突出关键信息

❌ 避免:
□ 大段落堆砌
□ 专业术语不解释
□ 长句子（超过 40 字）
□ 被动语态过多
```

## SEO 优化

### 关键词布局

```markdown
关键词位置优化:
1. 标题 (H1) - 必须包含主关键词
2. 第一段 - 前 100 字内出现关键词
3. 子标题 (H2/H3) - 自然融入关键词
4. 正文 - 关键词密度 1-3%
5. 结尾 - 再次提及关键词
6. 图片 alt - 描述性关键词

示例密度计算:
- 1000 字文章
- 主关键词出现 10-15 次
- 长尾关键词各出现 3-5 次
```

### 内链外链策略

```markdown
内链:
- 链接到相关历史文章
- 使用描述性锚文本
- 每篇文章 3-5 个内链

外链:
- 引用权威来源
- 数据/研究链接原始出处
- 避免链接到竞争对手
```

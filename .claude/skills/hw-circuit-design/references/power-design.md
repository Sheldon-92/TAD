# Power Design（电源设计）

电源方案设计 + 功耗预算 + 电池续航估算。流程：select → execute → verify → optimize。

## 1. Select — 电源架构

1. 输入源: USB 5V / 电池 3.7V / 太阳能 / 多源
2. 电压域识别：
   - 3.3V: MCU, 传感器, E-ink 逻辑
   - 1.8V: 某些传感器（如果需要）
   - VBAT: RTC 备用电池
   - 5V: USB 直供（充电 IC 输入）
3. 稳压方案选择：
   - LDO: 压差小（<1V）、电流小（<500mA）、噪声要求高
   - DCDC Buck: 压差大、电流大、效率优先
   - Charge Pump: 负压或升压小电流
4. 充电方案（如果电池供电）：
   - 线性充电: TP4056（简单，500mA-1A）
   - 开关充电: BQ24072（高效，支持路径管理）
5. 电池类型: LiPo / LiFePO4 / 锂亚 / 干电池
6. 画出电源拓扑图（D2 框图）

关键决策依据：效率 vs 成本 vs 噪声 vs 复杂度。
**质量门：电源架构决策必须基于负载需求计算，不是凭感觉。**
产出：power-architecture.md

## 2. Execute — 功耗预算计算

1. 列出所有模块的电流消耗（来自 datasheet），表格列：模块 / 模式 / 电压(V) / 电流(mA) / 占空比(%) / 平均电流(mA)。示例行（实际值必须来自 datasheet）：
   - ESP32-C3 Active TX: 3.3V, 350mA, 0.5% → 1.75mA 平均
   - ESP32-C3 Active RX: 3.3V, 80mA, 1.0% → 0.80mA
   - ESP32-C3 Modem Sleep: 3.3V, 15mA, 8.0% → 1.20mA
   - ESP32-C3 Deep Sleep: 3.3V, 0.005mA, 90.5% → 0.0045mA
   - E-ink Refresh: 3.3V, 40mA, 0.1% → 0.04mA；E-ink Idle: 0.001mA, 99.9% → 0.001mA
   - RTC Active: 3.3V, 0.002mA, 100% → 0.002mA
2. 计算系统级功耗：
   - 平均电流 = Σ(各模块平均电流)
   - 峰值电流 = Σ(各模块最大电流同时出现)
   - 考虑稳压器效率（LDO ~90%, DCDC ~85-95%）
3. 电池续航计算：
   - 续航时间(h) = 电池容量(mAh) / 平均电流(mA) × 0.8（降额系数）
   - 考虑自放电（LiPo ~3%/月）
   - 考虑温度影响（低温容量降低 10-30%）
4. 不同使用场景的续航估算：
   - 正常模式: 每小时唤醒一次发数据
   - 省电模式: 每天唤醒几次
   - 高频模式: 持续在线

**质量门：每个电流值必须标注 datasheet 来源。计算过程必须可复现。编造数据 = FAIL。**
产出：power-budget.md

## 3. Verify — 电源设计验证

1. 稳压器输出能力验证：
   - LDO 最大输出电流 > 峰值负载电流 × 1.5（安全余量）
   - DCDC 最大输出电流 > 峰值负载电流 × 1.3
   - 散热验证: P_dissipation = (Vin - Vout) × I_load, Tj < 125°C
2. 电池选型验证：
   - 电池容量 > 目标续航时间 × 平均电流 / 0.8
   - 电池放电倍率 > 峰值电流 / 容量
   - 电池尺寸是否装得下（物理约束）
3. 充电时间估算：充电时间(h) ≈ 电池容量(mAh) / 充电电流(mA) × 1.2
4. 电源上电时序验证：
   - MCU 在电源稳定后才释放复位
   - 外设在 MCU 初始化后才上电（如果有使能控制）
5. 异常场景：
   - USB 拔出时电池无缝切换
   - 电池低压保护触发电压 vs MCU 最低工作电压

**质量门：每个验证项必须有计算过程和 PASS/FAIL 结论。**
产出：power-verification.md

## 4. Optimize — 优化与报告

1. 优化方向：
   - 如果 LDO 压差大 → 考虑换 DCDC（提升效率 10-20%）
   - 如果 deep sleep 电流过大 → 检查哪个模块漏电流大
   - 如果续航不达标 → 调整唤醒策略或增大电池
   - 使能控制: 不用的模块关电源（GPIO 控制 LDO EN pin）
2. 生成功耗分布图：饼图（各模块功耗占比）、柱状图（不同模式电流对比）、时间线（典型工作周期电流波形）
3. 生成电源设计报告 PDF：电源拓扑图 + 功耗预算表 + 电池续航估算 + 优化建议

产出：power-report.pdf

## Quality Bar（pass/fail）

- 功耗预算覆盖所有模块的 active/sleep 电流
- 电流数据来自 datasheet（标注页码/表格编号）
- 电池续航计算包含降额系数（≥0.8）和温度影响
- 稳压器输出余量 ≥30%（峰值）
- 电源上电时序有明确描述
- 至少 3 种使用场景的续航估算
- 编造数据 = FAIL。每个电流值必须有 datasheet 来源。
